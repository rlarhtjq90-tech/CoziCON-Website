# 건설 입찰 플랫폼 개발 PLAN
**프로젝트**: 종합건설사 입찰공고 ↔ 전문건설사 입찰참여 B2B 플랫폼
**현재 상태**: 회원가입 / 로그인 구현 완료
**작성일**: 2026-05-03

---

## 1. 큰 그림 (Big Picture)

이 서비스는 단순한 게시판이 아니라 **B2B 거래 플랫폼**입니다. 따라서 일반 웹사이트와 달리 다음 3가지가 동시에 충족되어야 신뢰를 얻을 수 있습니다.

1. **사업자 신원 검증** — 가짜 업체, 페이퍼컴퍼니 차단
2. **거래 안전성** — 입찰 정보 보안, 마감 후 개찰, 분쟁 방지
3. **법적/규제 준수** — 건설산업기본법, 전자서명법, 개인정보보호법

핵심 사용자 흐름은 다음과 같습니다.

```
[종합건설사]                          [전문건설사]
   |                                     |
회원가입 → 사업자/건설업등록 인증 → 관리자 승인 → 로그인
   |                                     |
입찰공고 등록 ─────공고 게시판─────> 공고 검색/열람
   |                                     |
   |                              입찰서 제출 (견적+서류)
   |                                     |
마감 → 개찰 → 평가 → 낙찰자 선정 → 알림
   |                                     |
계약 → (선택) 정산/이행 단계
```

## 5. 비즈니스 모델 (수익화 옵션)

1. **종합건설사 구독제** — 공고 등록 무제한, 월정액
2. **전문건설사 구독제** — 공고 열람·입찰 권한 (가장 흔한 방식)
3. **건당 수수료** — 낙찰 금액의 일정 %
4. **프리미엄 노출** — 상단 배너, 추천 공고
5. **부가 서비스** — 전자계약, 보증보험 중개, 채권 매입

> 한국 건설업계 통념상 **전문건설사 멤버십 + 종합사 무료(또는 저가)** 모델이 가장 잘 작동합니다. 공고 수가 늘어야 전문사가 모이고, 전문사가 모여야 종합사가 들어오는 양면시장입니다. **초기에는 종합사 무료로 공급을 확보**하는 전략을 권장합니다.

## 8. MVP 권장 범위 (3개월)

자원이 한정적이라면 다음 범위를 MVP로 잡고 빠르게 시장 검증을 권장합니다.

1. 사업자/건설업 인증 + 관리자 승인
2. 공고 등록(첨부 포함) + 게시판 + 상세
3. 입찰서 제출 + 마감 자동 잠금
4. 개찰 + 낙찰자 선정 + 결과 알림
5. 이메일 알림, 기본 검색·필터
6. 간단한 관리자 페이지

이 범위로 **소수의 종합건설사를 직접 영입**해 진짜 공고를 올려보고, 전문사 반응을 보면서 Phase 4·5를 확장하는 것이 현실적입니다.

## 9. 흔히 놓치는 포인트 (전문가 코멘트)

1. **개찰 전 입찰가 비공개** — 종합사조차 마감 전엔 못 보게 해야 합니다. 담합·내부 유출 방지 필수. DB 컬럼 암호화 또는 별도 테이블 + 시간 기반 접근 제어.
2. **첨부파일 권한** — 도면은 자격조건 충족 회원에게만 다운로드 허용. Pre-signed URL + 만료시간.
3. **시간 동기화** — 마감 시각은 서버 시간 기준. 클라이언트 시계 신뢰 금지. 카운트다운만 클라이언트.
4. **감사 로그** — "누가 언제 무엇을 봤고/제출했는가"를 모두 기록. 분쟁 시 결정적.
5. **대용량 첨부파일** — 도면(특히 DWG) 100MB 이상 흔합니다. 직접 업로드(서버 경유) 대신 S3 Multipart Upload.
6. **입찰 무효 처리** — 자격 미달 발견 시 사후 무효 처리 + 차순위 낙찰 절차.
7. **세금계산서·정산** — 처음부터 PG·세금계산서 자동화를 붙이지 말고, 결제 발생할 때 도입.
8. **알림톡** — 한국에서는 이메일보다 카카오 알림톡 도달률이 압도적. 마감 임박 알림은 알림톡으로.
9. **블랙리스트** — 부도·체불 이력 업체 차단 메커니즘.
10. **양면시장 부트스트래핑** — 닭과 달걀 문제. 처음에는 한 쪽(보통 공급=공고)을 무료/지원금으로 채우고, 다른 쪽(수요=전문사 멤버십)에서 수익 발생시키는 전략.
