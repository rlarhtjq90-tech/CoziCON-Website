## 3. 페이지 설계

### 3-1. `/verify-license` — 건설업등록증 인증

- 페이지 진입 시 세션의 `bizNo` + `userType`으로 KISCON API(`/api/verify-license`) 자동 호출
  - `GENERAL_CONTRACTOR` → `licenseType: 'general'`
  - `SPECIALTY_CONTRACTOR` → `licenseType: 'specialty'`
- 조회 성공: 면허 목록 표시 (업종명, 면허번호, 등록일) + 체크박스로 등록할 면허 선택
- 조회 실패 / API 키 없음: "면허를 찾을 수 없습니다" 안내 + 파일 업로드만으로 진행 가능
- 건설업등록증 파일 업로드 (선택, JPG·PNG·PDF 10MB 이하)
- "인증 완료" 클릭 → `/api/license/verify` POST → License DB 저장 → `/dashboard` 이동

### 3-2. `/admin` — 관리자 승인 큐

- 접근 제한: `ADMIN_EMAILS` env var에 등록된 이메일만 허용, middleware에서 차단
- `User.status = PENDING` && `companyId != null` 인 사용자 목록 표시
- 표시 정보: 회사명, 사업자번호, 회원 유형, 신청일, 업로드된 서류 링크
- 승인 버튼 → `User.status = ACTIVE`
- 반려 버튼 → `User.status = REJECTED`
- 처리 후 목록에서 해당 항목 제거 (페이지 새로고침 없이 낙관적 업데이트)

## 4. API 설계

### `POST /api/license/verify`
- 인증 필요 (로그인)
- Body: `{ selectedLicenses: LicenseItem[], licenseDocUrl?: string }`
- 동작: License 레코드 upsert (companyId 기준), 파일 URL 저장
- 응답: `{ success: true }`

### `POST /api/admin/approve`
- 인증 필요 + ADMIN_EMAILS 체크
- Body: `{ userId: string }`
- 동작: `User.status = ACTIVE`
- 응답: `{ success: true }`

### `POST /api/admin/reject`
- 인증 필요 + ADMIN_EMAILS 체크
- Body: `{ userId: string }`
- 동작: `User.status = REJECTED`
- 응답: `{ success: true }`

## 5. DB 변경

없음. 기존 `License` 테이블 사용.

```prisma
model License {
  id          String    @id @default(cuid())
  companyId   String
  licenseType String    // 업종명 (예: "토목공사업")
  grade       String?
  licenseNo   String?   // 면허번호
  issuedAt    DateTime?
  isActive    Boolean   @default(true)
  createdAt   DateTime  @default(now())
}
```
